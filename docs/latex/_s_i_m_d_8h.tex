\doxysubsection{Detailed Description}
Architecture-\/agnostic SIMD abstractions and feature detection macros. 

\begin{DoxyAuthor}{Author}
Alan Abraham P Kochumon 
\end{DoxyAuthor}
\begin{DoxyDate}{Date}
Created on:\+ March 07, 2026
\end{DoxyDate}


\begin{DoxyParagraph}{Configuration}
Define FORCE\+\_\+\+SSE to turn on SSE, even if hardware supports newer instructions Similar options include FORCE\+\_\+\+AVX512, FORCE\+\_\+\+AVX2, FORCE\+\_\+\+SSE, and FORCE\+\_\+\+SCALAR which will completely turn off SIMD. 
\end{DoxyParagraph}
\begin{DoxyNote}{Note}
Even if flags are specified, it will only default to the max that the CPU architecture supports
\end{DoxyNote}
\begin{DoxyParagraph}{Compiler note}
Make sure compiler flags are turned on:\+ -\/mavx2 (GCC/\+\+Clang) or /\+arch:\+AVX2(\+MSVC) for turning on AVX2 support -\/mavx512 (GCC/\+\+Clang) or /\+arch:\+AVX512 (MSVC) for turning on AVX512 supported. 
\end{DoxyParagraph}
\begin{DoxyNote}{Note}
If you compile using the supplied CMake\+List.\+txt, then, it will turn on the flag supported by the CPU.
\end{DoxyNote}
